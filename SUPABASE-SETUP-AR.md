# إضافة التخزين السحابي (Supabase) لـ LCare IQ

هذا الدليل يربط تطبيقك بقاعدة بيانات سحابية تجعل **كل الأجهزة تشارك نفس البيانات** (إنت + الطبيبة + المساعدة)، بدل التخزين المحلي المنفصل.

---

## ما الذي تغيّر في الكود

تم تحويل طبقة التخزين لتعمل سحابياً تلقائياً:
- البيانات تُحفظ محلياً فوراً (سرعة) **و** تُرفع للسحابة بالخلفية
- عند فتح التطبيق على أي جهاز، يسحب أحدث نسخة من السحابة
- التغييرات تظهر فوراً على كل الأجهزة (مزامنة حيّة)
- تظهر شارة **☁️ مُزامَن** أسفل الشاشة عند نجاح الاتصال

كل بيانات المرضى والجلسات والتحاليل والقراءات والمستخدمين تتزامن — بدون أي تغيير في طريقة استخدامك للتطبيق.

---

## الخطوات (مرة واحدة فقط)

### ١. أنشئ جدول قاعدة البيانات

> لديك مشروع Supabase جاهز بالفعل.

1. ادخل [supabase.com](https://supabase.com) → افتح مشروعك
2. من القائمة الجانبية: **SQL Editor** → **New query**
3. افتح ملف `supabase-setup.sql` (المرفق)، انسخ محتواه كاملاً، الصقه
4. اضغط **Run** (أو Ctrl+Enter)
5. لازم تظهر رسالة نجاح (Success)

### ٢. انسخ مفتاحي مشروعك

من Supabase: **Settings** (الترس) → **API**

انسخ قيمتين:
- **Project URL** — مثل `https://abcdxyz.supabase.co` (بدون `/` في النهاية)
- **anon / public** key — مفتاح طويل يبدأ بـ `eyJ...` (هذا آمن للمتصفح)

> ⚠️ لا تستخدم مفتاح `service_role` السري إطلاقاً — فقط `anon / public`.

### ٣. ضع المفتاحين في الكود

افتح ملف `index.html`، في الأعلى ابحث عن هذا الجزء (سطر ~135):

```javascript
window.__LCARE_CONFIG = {
  supabaseUrl:     "PUT_YOUR_SUPABASE_URL_HERE",
  supabaseAnonKey: "PUT_YOUR_SUPABASE_ANON_KEY_HERE",
  workspaceId:     "lcare-main"
};
```

بدّل القيمتين بقيم مشروعك:

```javascript
window.__LCARE_CONFIG = {
  supabaseUrl:     "https://abcdxyz.supabase.co",
  supabaseAnonKey: "eyJhbGciOiJIUzI1NiIs...",
  workspaceId:     "lcare-main"
};
```

> اترك `workspaceId` كما هو (`lcare-main`) — هو ما يجعل كل أجهزتك تشترك بنفس البيانات.

### ٤. ارفع التحديث على GitHub

استبدل ملف `index.html` على مستودعك (نفس طريقة المرات السابقة: Edit → الصق → Commit).

### ٥. انتظر النشر واختبر

1. انتظر Netlify ينشر (دقيقة-دقيقتين)
2. افتح موقعك → لازم تشوف شارة **☁️ مُزامَن** أسفل اليسار
3. أضف مريض تجريبي
4. افتح الموقع على جهاز ثاني (أو متصفح آخر) → لازم يظهر نفس المريض ✅

## ملاحظات مهمة

**الأمان:** المفتاح `anon` آمن للمتصفح بحكم تصميمه، ومحمي بـ Row Level Security الذي فعّلناه في SQL. هذا مختلف عن مفتاح Anthropic (الذي يبقى على الخادم).

**الترقّي (migration):** أول مرة تفتح التطبيق بعد الربط، إذا كان عندك بيانات محلية موجودة، سترفع تلقائياً للسحابة. وإذا السحابة فيها بيانات، ستُسحب للجهاز.

**النسخ الاحتياطي:** ما زال بإمكانك تصدير JSON من داخل التطبيق كنسخة احتياطية إضافية.

**التكلفة:** خطة Supabase المجانية تكفي لآلاف المرضى ومئات آلاف القراءات.

---

## إن لم تظهر شارة "مُزامَن"

- تأكد أنك بدّلت القيمتين بشكل صحيح (بدون مسافات زائدة، الـ URL بدون `/` في النهاية)
- تأكد أنك شغّلت `supabase-setup.sql` بنجاح
- افتح Developer Tools (F12) → Console → ابحث عن رسائل خطأ تبدأ بـ `cloud` أو `Supabase`
- تأكد أنك أعدت النشر بعد تعديل `index.html`
